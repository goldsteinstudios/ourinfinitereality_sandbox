## 1. `just_math_v7.5.md` contradicts itself

Arithmetic, not interpretation. Substituting `X = Y` into `XY = 1ₙ` gives `X = Y = ±√1ₙ`, so
`Gₙ ∩ Bₙ` is **two** points, `(√1ₙ, √1ₙ)` and `(−√1ₙ, −√1ₙ)` in `(X,Y)`. Their midpoint is `(0,0)`.

- **Prop. 3.1** (line 51) names `P = (√1ₙ, √1ₙ)`. It lists only one of the two intersection points.
- **Lemma 4.4** (line 85) says "the straight segment from `+1` to `−1` passes through `P` (the two
  modes are **antipodal about the center**)".

A pair whose midpoint is `(0,0)` cannot be antipodal about `(1,1)`. `P` is used for the intersection
point and for the origin within one argument. In the `(a,b)` chart the same two loci are `(±√1ₙ, 0)`,
since `b = (X−Y)/2 = 0` wherever `X = Y`.

v7.6c parallax r1 logs this independently, corrections-ledger entry 4:

## 2. Math r5's repair overshoots: it seats the modes at the balance point

r5 correctly separates `Oₙ = xaxisₙ ∩ yaxisₙ` (the origin) from `Pₙ = Gₙ ∩ Bₙ`. It then declares

  `X = Y = √1ₙ` at `Pₙ`, so `XY = 1ₙ ≠ 0`: the *conserved product* is intact. What vanishes there is

  and its conclusion does not follow** — `X = Y` does not drive `XY` toward `0`, so occupation of `P`

  variables `Xₙ, Yₙ`. r5 only made the identification explicit enough to be caught.

- The modes *as directions* are the asymptotes `Xₙ → 0ₙ`, `Yₙ → 0ₙ`, which r5 itself calls
  **unreachable**. There is no seat anywhere on the structure.

- r5's recursion clause says the child's `O₍ₙ₊₁₎` **is** the parent's `Pₙ` — "the same locus" — and
  its Notation calls every `Oₙ` unoccupiable. One locus, both occupied and unoccupiable. r5 escapes
  only by making occupancy frame-relative: *"unoccupiable origin from within, limit-position from
  without."*

- r5 flags the step itself: *"Commitment, flagged: the conjugate mode is **seated** as a point of
  `Gₙ`… **Theorems 1–3 depend on the seating.** [commitment]"* Its `aperture` section is a patch over

## 5. The type discipline

Four distinct types, currently merged in both doc sets:

| Type | Objects |
|---|---|
| Coordinate axes (loci) | `xaxisₙ = {Yₙ = 0ₙ}`, `yaxisₙ = {Xₙ = 0ₙ}` — the asymptotes of `Gₙ` |
| Mode variables | `Xₙ`, `Yₙ` — **not loci, not points** |
| Loci | `Gₙ = {XₙYₙ = 1ₙ}`, `Bₙ = {Xₙ = Yₙ}`, `Mₙ = {a = 0}` |
| Algebra elements | `±1`, `±i` — elements of the orbit algebra |

r5's Notation writes `Oₙ = Xₙ ∩ Yₙ`. Two variables cannot be intersected; it means
`xaxisₙ ∩ yaxisₙ`. Small on the page, consequential downstream: it is what lets `Xₙ` slide from
mode variable → axis → the point `+1`, which is the seating.

### Consequence A — `O₍ₙ₊₁₎ = Pₙ` is a theorem, not a referent rule

Both doc sets already fix the axis mapping: `Gₙ → xaxis₍ₙ₊₁₎` (r5: *"the tangent structure at `Pₙ`"*),
`Bₙ → yaxis₍ₙ₊₁₎`. Apply the definition of an origin to the child:

```
O₍ₙ₊₁₎ = xaxis₍ₙ₊₁₎ ∩ yaxis₍ₙ₊₁₎ = Gₙ ∩ Bₙ = Pₙ
```

Physics r3's "referent discipline" and parallax r1's through-line 3 both hand "every `Oₙ` is some
parent's `Pₙ`" down by fiat. It is a one-line consequence of the axis mapping.

### Consequence B — ~~every locus on the generative line is forbidden twice~~ **WITHDRAWN**

~~In frame `n`, `Pₙ` is forbidden because `Xₙ = Yₙ` (the saturation face `|1|`); in frame `n+1` the
same locus is `O₍ₙ₊₁₎`, forbidden because both coordinates vanish (the absence face `|0|`).~~

**Withdrawn on two counts.**

1. The premise is the struck argument from §2 — `Xₙ = Yₙ` does not cancel the conserved product.
2. Ledger C3 says the locus is **realized in the parent's expression** and unmet in the child's.
   *Realized is not forbidden.* There are not two prohibitions; there is one locus with two
   frame-readings. And only `Oₙ` is called a local face of `P₀` — `Pₙ` is not.

The `|0|`/`|1|` face reading is not merely unruled. It is probably wrong. It is recorded here only so
it is not silently reused.

### Consequence C — "asymptote" vs "symmetry axis" is frame-relative, like curvature

- **In frame `n`:** `Gₙ` is curved, `Bₙ` is straight. `Bₙ` is `Gₙ`'s orthogonal **symmetry axis** —
  the fixed locus of `σ_B : X ↔ Y`. `Bₙ` crosses `Gₙ` at `Pₙ`; it is not asymptotic to it.
- **In frame `n+1`:** both are straight axes, and both are **asymptotes** of `G₍ₙ₊₁₎`. The asymmetry
  is gone.

Only the **tangent** to `Gₙ` at `Pₙ` is inherited, so `Gₙ`'s global curvature is exactly what does
not cross into the child. Hence the parent's *local* ⊥ at `Pₙ` becomes the child's *global* ⊥ —
already stated at `parallax_v7.5.md:51`.

**Corollary.** r5's aperture result — perpendicularity of `Gₙ`'s tangent to `Bₙ` holds iff `Y = X`,
i.e. at `Pₙ` and nowhere else — is not a patch on the persistence problem. It is the statement that
the child's chart inherits a one-point property.
